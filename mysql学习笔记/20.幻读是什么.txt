1.for update 语句 是在这一行加上写锁，其他事务不能修改该行，隔离级别是【当前读】

2.幻读：是指在一个事务中，前后两次相同的查询，查询出来的数据不一致，后一次看到了比前一次多的数据(幻读只会出现在【当前读】)

3.幻读造成的问题，举例: sessionA: 开启事务 select * from t wehre d=5 for update; update t set d= 100 where d = 5;
					  sessionB: insert into t values(1,1,5); update t set c = 5 where id = 1;
     				  sessionA:提交事务
     				  造成的问题是，id为1的哪一行 d的值也会被改成100

4.为解决这个问题产生间隙锁（只出现在【可重复读】隔离级别下）:锁住行与行之间的间隙，这就确保这个时候不能有新记录插入进来
	间隙锁只会去阻塞往间隙插入数据 间隙锁会影响系统的并发性，因为两个不同事务的间隙锁会锁死另一个的插入操作

5.行锁和间隙锁合称为next-key lock (前开后闭的)
