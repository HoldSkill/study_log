1.对索引字段做函数操作，有可能会破坏索引值的有序性，因此优化器放弃使用该字段的索引
	比如 month()函数

2.对于那些不会破坏索引有序性的函数，优化器也不会选择使用索引，如select * from xxx where id +1 = 10000，对于这种情况要根据实际场景改为 select * from xxx where id = 10000-1

3.隐式类型转换也不会用到索引
	1).在mysql中，字符串和数字作比较，字符串会转换成数字，所以，如果一个字段是字符串类型的时候，如果搜索的值是整形，就会用转换类型的函数，把该字段转换成整形，所以也用不到该字段的索引
	2).联表时，如果被驱动表(explain中第二张或者后面的表)的字符集比驱动表的字符集小,就需要转换成驱动表的字符集(如：驱动表 utf8mb4 ,被驱动表 utf8)，所以也不会用到索引
